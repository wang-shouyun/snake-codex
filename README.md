# Snake Legends

一个使用 **Python + pygame** 开发的多模式贪吃蛇游戏项目。  
它不仅包含传统贪吃蛇玩法，还加入了 **AI 对战、局域网联机、难度系统、节奏模式、策略模式、道具系统、统计存档、多语言界面** 等高级功能。

这个项目适合：

- Python 初学者练手
- pygame 入门学习
- 课程设计 / 作品集展示
- 想系统学习“如何把一个小游戏做成完整项目”的同学

## 项目简介

`Snake Legends` 是一个结构化、多文件拆分的贪吃蛇项目。  
它的目标不是只做一个“能运行”的小游戏，而是做成一个：

- 结构清晰
- 适合初学者阅读
- 方便后续扩展
- 可以继续加功能和模式

的完整游戏项目。

这个项目目前已经包含：

- 基础贪吃蛇玩法
- 多种游戏模式
- 特殊食物和状态效果
- 多语言菜单
- 人机对战
- AI 自动演示
- 局域网主机 / 加入
- JSON 本地数据保存
- PyInstaller 打包支持

## 项目亮点

和普通贪吃蛇相比，这个项目的亮点主要有：

- **玩法不止单人**
  除了单人，还有双人、本地对战、合作模式、局域网对战。

- **AI 不是随机乱走**
  AI 会找路径、避险、保命、评估空间。

- **不只是“吃了加分”**
  项目包含节奏模式、策略模式、技能点、事件分支等玩法。

- **代码不是一个大文件堆在一起**
  项目被拆分为多个清晰模块，适合阅读、维护和扩展。

- **不只是开发环境能跑**
  已支持通过 PyInstaller 打包成 Windows 独立应用。

```powershell
python --version
```

## 零基础运行教程

如果你是第一次接触 Python 游戏项目，照下面步骤做就可以。

### 第一步：安装 Python

去 Python 官网下载安装 Python 3.11 或 3.12。  
安装时请勾选：

```text
Add Python to PATH
```

否则你在终端里输入 `python` 可能会提示找不到命令。

---

### 第二步：下载项目

#### 方法 1：直接下载 ZIP
在 GitHub 页面点击：

```text
Code -> Download ZIP
```

下载后解压即可。

#### 方法 2：使用 Git
如果你装了 Git，可以使用：

```powershell
git clone 你的仓库地址
```

---

### 第三步：进入项目目录

假设你的项目文件夹叫 `snake-codex`，进入：

```powershell
cd snake-codex\snake_game
```

---

### 第四步：安装依赖

本项目目前最重要的依赖是 `pygame`：

```powershell
pip install pygame
```

如果你用的是虚拟环境，也可以这样安装：

```powershell
python -m pip install pygame
```

---

### 第五步：运行项目

在 `snake_game` 目录下运行：

```powershell
python main.py
```

如果你使用仓库自带虚拟环境，可以这样运行：

```powershell
..\.venv\Scripts\python.exe main.py
```

---

### 第六步：开始游戏

程序启动后流程是：

1. 进入语言选择页
2. 进入主菜单
3. 选择模式
4. 进入开始设置页
5. 设置游戏时长
6. 选择“直接开始”或“AI 演示”
7. 正式进入对局

## 使用到的模块和技术

这个项目主要使用了这些 Python 标准库和第三方库。

#### `pygame`
这是整个项目最核心的库，用来做：

- 创建窗口

## 各模式说明

### 1. Timed Solo
单人限时冲分模式。

特点：
- 只有一个玩家
- 时间可自定义
- 适合练手和刷分
- 蛇越长速度越快

适合谁：
- 想练方向控制的人
- 想测试难度差异的人
- 想快速体验项目的人

---

### 2. Human Vs AI
玩家对 AI 的模式。

特点：
- 玩家与 AI 同场竞争
- AI 会自己找路吃食物
- AI 会规避危险
- 可以观察 AI 路径和决策状态

适合谁：
- 想挑战 AI
- 想研究 AI 行为
- 想看人机对抗效果

---

### 3. Local Duel
本地双人对战。

特点：
- 两个人同一台电脑玩
- 玩家一用方向键
- 玩家二用 WASD
- 双方同场竞争

适合谁：
- 和朋友面对面对战
- 做双人游戏演示

---

### 4. Co-op Run
双人合作模式。

特点：
- 两人一起达成团队目标
- 不再是单纯竞争
- 更强调配合

适合谁：
- 想体验合作玩法
- 想让双人模式不只是打架

---

### 5. LAN Host
局域网主机模式。

特点：
- 创建房间
- 显示主机 IP
- 显示房间码
- 等待其他电脑加入

适合谁：
- 局域网展示
- 教室 / 寝室 / 办公室内测试

---

### 6. LAN Join
局域网加入模式。

特点：
- 可以输入主机 IP
- 也可以从自动发现列表加入
- 加入后接收主机快照

---

### 7. Rhythm Mode
节奏模式。

特点：
- 蛇移动与节拍同步
- 食物刷新与节拍同步
- 有连击系统
- HUD 会显示当前节拍和连击
- 支持 `assets/music/` 中的音乐文件

适合谁：
- 想学节拍驱动逻辑的人
- 想做音乐小游戏扩展的人

---

### 8. Strategy Mode
策略模式。

特点：
- 每条蛇有技能点
- 吃食物或道具可以获得技能点
- 存在技能卡
- 有随机事件和分支选择
- AI 行为会根据策略偏向调整

适合谁：
- 想让贪吃蛇更像策略游戏
- 想继续做高级玩法扩展

## 难度系统说明

项目支持三档难度。

### Easy
特点：
- 蛇移动更慢
- 普通食物刷新更慢
- 特殊食物更保守
- 障碍更少

适合：
- 新手
- 演示
- 教学

### Normal
特点：
- 默认平衡模式
- 速度和障碍都适中

适合：
- 普通体验
- 大多数玩家

### Hard
特点：
- 蛇移动更快
- 特殊食物更频繁
- 障碍更多

适合：
- 想挑战更高难度的人

## AI 系统说明

这个项目的 AI 不是简单随机走，而是带逻辑判断的。

### AI 做了什么
- 会优先寻找目标食物
- 会评估路径是否安全
- 会判断吃到食物后还能不能脱困
- 会避开危险头位
- 会在不安全时优先找更大的可生存空间

### AI 用到的思路
- BFS 广度优先搜索
- 危险区域规避
- 可达空间评估
- 保命优先 fallback

### 对初学者有什么学习价值
你可以从这里学到：

- 路径搜索怎么写
- AI 如何避免自杀
- 游戏 AI 并不一定要“特别复杂”
- 先做“稳定 AI”，再做“强 AI”

---

## 节奏模式说明

节奏模式的重点是：**把游戏逻辑和音乐节拍绑定**。

### 核心思路
- 先设定 BPM
- 根据 BPM 算出每拍间隔
- 每到节拍，允许移动
- 每隔若干拍，刷新食物
- 吃食物时判断是 Perfect / Good / Miss
- 更新连击数和最佳连击

### 这部分适合学习什么
- 时间驱动逻辑
- 音乐节拍同步
- 连击系统
- HUD 实时信息更新

### 音乐文件放哪里
```text
assets/music/
```

支持：
- `.ogg`
- `.mp3`
- `.wav`

---

## 策略模式说明

策略模式的目标是让贪吃蛇不再只是“吃和躲”。

### 技能点系统
每条蛇都可以有技能点。

技能点获取方式：
- 吃食物
- 触发道具
- 事件奖励

---

### 技能卡系统
策略模式中支持技能卡，例如：
- Dash
- Shield
- Magnet
- Jammer

技能卡需要消耗技能点使用。

---

### 随机事件系统
游戏过程中可能随机触发事件，例如：
- Resource Cache
- Storm Front
- Intel Drop

每个事件都可能有不同分支选择。

---

### AI 策略偏向
策略模式下，AI 不再只追求最近食物，还会根据偏向改变行为：

- `aggressive`
  更激进，更爱抢食物

- `safe`
  更保守，更看重空间和存活

- `balanced`
  均衡玩法

---

## 局域网联机说明

本项目支持局域网联机。

### 主机流程
1. 进入 `LAN Host`
2. 游戏显示本机 IP 和房间码
3. 等待另一台电脑加入

### 客户端流程
1. 进入 `LAN Join`
2. 输入主机 IP
3. 或在自动发现列表中选择房间
4. 成功连接后进入联机对局

---

### 联机底层做法
- 使用 `socket`
- TCP 负责主机和客户端通信
- UDP 广播负责房间发现
- 主机为权威端
- 客户端发送输入
- 主机发送状态快照

---

### 为什么这样设计
对于初学者来说，这是最容易理解的一种联机结构：

- 主机负责计算
- 客户端只负责输入
- 状态统一由主机同步

这样逻辑最清楚，也最稳定。

## 常见问题

### 1. 运行时报 `No module named pygame`
说明你没有安装 pygame。

解决方法：

```powershell
pip install pygame
```

---

### 2. 中文显示异常
项目已经优先查找支持中文的系统字体。  
如果个别电脑仍然异常，通常是系统缺少中文字体。

---

### 3. 局域网找不到房间
请检查：

- 两台电脑是否在同一局域网
- 防火墙是否拦截端口
- 主机是否已成功开房
- 输入的 IP 是否正确

---

### 4. 音乐不播放
请检查：

- `assets/music/` 中是否真的有音频文件
- 格式是否为 `.ogg` / `.mp3` / `.wav`
- 本机音频驱动是否正常

---

### 5. 游戏数据保存在哪里
默认保存在：

```text
data/player_stats.json
```
